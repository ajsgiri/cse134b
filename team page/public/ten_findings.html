10 Findings

Validate
  1. <bdi> elements are not supported in every browser so we get a warning from the validator. Through more research we found that internet explorer does not support this feature. Another way to do this is through css of unicode-bidi
  2. Date, time, and color input types are not supported in every browser so the validator gave us a warning. We found that the date type input is inconsistent among different browsers so developers have to pay attention to the format of the date. This is also the case with time and color.
  3. Keygen is deprecated and may be no longer supported at anytime. It is an obsolete element.
  4. onMouseWheel is deprecated. It is recommended to use the onwheel attribute instead. Onmousewheel event is inconsistent among browsers so the cross-browser compatibility is poor.
  5. The validator found that prefetching with an anchor tag was invalid even though the w3 website said that it was. Through further research it was found on MDN that only link supports the rel="prefetch" value. To remediate this, we used a link tag to prefetch the url so that it is in cache, and then we added an anchor link to hyperlink to the site.
  6. MathML is only supported by firefox and safari. However on chrome, even though it says it is not supported, it looks like it renders normally.
  7. The strike, baseline, nobr, and center elements are obsolete and all of the suggestions is to use CSS properties
  8. Spellcheck only works when the text can be edited, so it will only show when we use contenteditable.
  9. The lang tag seems to serve more purpose to the coder than the user. When we use the lang tag, it doesn't explicitly specify to the user the language that it is in, it will only be clear to whoever is editing or looking at the code.
  10. Translate is new to HTMl5, it is used to explicitly indicate which elements should or should not be translated. Even though it passed the validation test, it is currently not supported on Chrome, IE, FireFox, Safari, and Opera.
